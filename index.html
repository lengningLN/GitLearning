I am learning Git （2019-05-06）

Terminologys (术语)

  1. git的三大类型对象

    commit : 每次提交的对象

    tree : 文件夹

    blob : 指具体的某个文件,blob和文件名没有关系，只要文件内容相同，就是同一个blob

Basic Commands (常用命令)

  1. 常用的Linux命名

    1. 创建文件夹：mkdir 文件夹名称

    2. 新建一个文件并写入一段话：echo '文件内容' > 文件名

    3. 移动文件：mv 原地址/文件名 新地址/文件名

    4. 查看当前目录：pwd

    5. 查看当前路径下的所有文件的详细信息，包括隐藏文件：ls -al

  2. 探索.git目录

    HEAD : 指向当前分支的引用

    config : git仓库的配置文件，以及local的用户信息存放在这里；

    refs : 引用，里面存放tags(里程碑)和heads(分支)

    objects : 存放所有的git对象

  3. 添加配置

    git config --local/global/system user.name 'Your name'

    git config --local/global/system user.email 'Your email'

    清除，--unset

    git config --unset --local/global/system user.name

    查看配置

    git config --list --local/global/system

  4. 基本操作

    git status : 查看工作区、暂存区的状态

    git add 文件名: 将工作区新建/修改的内容添加到暂存区

    git commit -m "commit message" 文件名: 将暂存区的内容提交到本地库

    git branch newBranchName : 创建新分支

    git checkout newBranchName : 切换到新分支

    git merge newBranchName : 将newBranchName合并到当前分支

    git branch -d newBranchName : 删除newBranchName分支

    git status : 查看当前文件的状态，如果合并出现冲突，使用此命令可以显示冲突的文件

  5. 查看历史

    git log --all : 查看所有分支历史

    git log --all --graph : 查看图形化历史

    git log --oneline : 查看单行的简洁历史

    git log --oneline -n3 : 查看最近三条的简介历史

    git log --oneline --all -n3 --graph : 查看所有分支最近4条单行的图形化历史(log 之后的命令没有先后顺序，可任意排序)

  6. 查看对象内容 git cat-file : 显示版本库对象的内容、类型以及大小信息

    -p : 根据对象的类型，以优雅的方式显示对象的内容

    -t : 显示对象的类型

    -s : 显示对象的大小

  7. 分支操作
  8. 比较文件差异

    git diff [文件名] : 将工作区中的文件和暂存区的进行比较

    git diff [本地库历史版本] [文件名] : 将工作区的文件和本地库历史记录比较，不带文件名的话，会比较多个文件

    git diff 一个commit的hash版本号 另一个commit的hash版本号 ： 比较两个版本的不同之处

    git diff HEAD HEAD~1 (或者git diff HEAD HEAD^ ) : 比较最近两个版本的异同

    ^和~都是父节点，区别是跟数字的时候，^2是第二个父节点，~2是父节点的父节点。^和~可以组合使用，比如：HEAD~2^2 标示的是当前节点的父节点的父节点的第二个父节点

  9. 解决冲突

    1. 编辑文件，删除特殊符号，删除重复内容或者保留内容

    2. 将文件修改完毕后，保存退出。

    3. git add [文件名]

    4. git commit -m"日志信息" (注意此时commit时不能带文件名)
